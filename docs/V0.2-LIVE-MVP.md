# Pathoro v0.2 — Live MVP

**Live URL:** https://pathoro-mvp.vercel.app

## v0.2 status

- Live on Vercel
- Production verified

## What changed from v0.1

- Five direction questions ([`docs/V0.2-DIRECTION-QUESTIONS.md`](V0.2-DIRECTION-QUESTIONS.md)), replacing the fixed demo orientation with a lightly personalized guided flow
- Suggestion chips for each question, so users can pick a starting answer instead of writing one from scratch
- Custom answer editing with a pencil affordance, so it's clear each answer can be picked from a suggestion or written by hand
- `localStorage` persistence — answers are saved on every edit and survive a page refresh
- Question 4 ("What would make this feel more reachable?") maps to the selected route on `/route-planning`, with a manual route click overriding the mapped selection for that page session

## Current limitations

- Answers are local/browser-only — no account, no cross-device sync, cleared if browser storage is cleared
- Opportunities are still mocked — a single hardcoded opportunity (Plant-Based Cooking Class) regardless of what a user enters
- No backend, auth, AI, or opportunity posting yet

## Current product loop

Direction questions → route selection → opportunity detail → next step

See [`V0.1-LIVE-MVP.md`](V0.1-LIVE-MVP.md) for the original three-screen flow and [`MVP-LOCKED-PRINCIPLES.md`](MVP-LOCKED-PRINCIPLES.md) for the product principles this MVP is designed around but does not yet fully implement.
