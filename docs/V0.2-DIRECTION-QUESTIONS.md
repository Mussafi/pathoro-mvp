# Pathoro v0.2 — Five Direction Questions

This is the first implementation of the [Five Direction Questions](MVP-LOCKED-PRINCIPLES.md#five-direction-questions) locked product principle. It turns the previously fixed demo into a lightly personalized guided orientation flow, still using only mock data and client-side storage.

## The five questions

Asked on the homepage, in the orientation section, with sensible pre-filled defaults:

1. What are you trying to move toward?
2. What should this make room for?
3. Where are you starting from?
4. What would make this feel more reachable?
5. What could you try first?

## What each question maps to

| Question | Maps to |
|---|---|
| What are you trying to move toward? | Direction |
| What should this make room for? | Meaning / fuel |
| Where are you starting from? | Current location |
| What would make this feel more reachable? | Route type / support need |
| What could you try first? | First movement |

## How it works

- Answers are editable inline on the homepage orientation card (`components/OrientationCard.tsx`), pre-filled with defaults so the existing demo still works untouched.
- Answers are saved to `localStorage` on every edit (`lib/direction.ts`, `lib/useDirectionAnswers.ts`) — no database, no account, no server round-trip.
- On `/route-planning`, the left rail (`PathSidebar`) and the route context panel (`RouteExplorer`'s "Best next route" aside) read the saved answers and display them: direction, what it should make room for, where the user is starting from, what would help, and what they said they'd try first.
- Real Openings Route remains selected by default regardless of the answers — this version personalizes the surrounding text only, it does not change which route is recommended.
- Screen 3 (`/opportunity/plant-based-cooking-class`) is unchanged. It still shows the single mocked opportunity.

## What is still mocked

- The five answers only affect display text on Screen 2 — they do not change which routes, steps, or opportunities are shown.
- There is a single hardcoded opportunity (Plant-Based Cooking Class) regardless of what a user enters.
- Answers persist only in the current browser's `localStorage`. Clearing browser storage or switching devices resets to the defaults.
- No validation, no character limits, no guidance on what makes a "good" answer.

## What will later require real AI / opportunity data

- Generating a different Best Next Route recommendation based on the "What would make this feel more reachable?" answer.
- Generating or selecting a real, relevant opportunity based on "What are you trying to move toward?" instead of always showing the cooking class.
- Turning "What could you try first?" into an actual actionable first step tied to a real opportunity, rather than a restated answer.
- Saving answers to an account so they persist across devices and sessions.
