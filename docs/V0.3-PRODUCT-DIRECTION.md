# Pathoro v0.3 — Guided, Local, Magical

## v0.3 goal

Make the product feel more guided, more local, more magical, and more alive — without adding a backend, auth, Supabase, or real AI. Everything in this version is still mock data and `localStorage`.

## The five improvements

### A. A more guided five-question flow

The five direction questions, suggestion chips, pencil edit affordance, and chevron suggestions are unchanged. On top of that:

- When a question card is expanded, it shows "Question X of 5" so the flow feels like a guided sequence rather than a flat form.
- Only one question is expanded at a time (unchanged from v0.2, preserved here).
- A closing line after the five questions — "Pathoro will use this to suggest a route and nearby opportunities." — connects the questions to what happens next.
- The top reassurance ("Choose a suggestion or write your own.") is unchanged.

### B. Route selection feels suggested, not random

On `/route-planning`, when the currently selected route matches the route Question 4 maps to, the detail panel shows:

- A "Suggested from your answers" badge with a soft glowing sparkle icon
- A short explanation naming the answer that produced the suggestion, e.g. *"You chose 'Someone who has done it before,' so Pathoro surfaced the People Route first."*
- A subtle green glow behind the star badge on the selected route in the route list

If the user clicks a different route manually, that override takes over as the selected route for the rest of the page session (unchanged from v0.2) and the "Suggested from your answers" framing only reappears if the manually chosen route happens to match the mapped one.

### C. Multiple opportunity examples

A new mock opportunity data model ([`lib/opportunities.ts`](../lib/opportunities.ts)) with one example per route. `/route-planning` shows an "Opportunities from this route" section with cards for the currently selected route, plus a "Coming soon" filler card so the section always shows more than one tile.

### D. "Post an opportunity" preview

A card on `/route-planning` — "Have an opportunity to offer? Help someone find their next step." — opens a preview modal (not a real form submission) explaining that opportunity posting is coming soon, with four disabled preview fields.

### E. Local opportunity language

A "Looking near" location field, defaulting to Austin, TX, lives on the homepage's five-question card and is stored alongside the direction answers. It shows up as local language throughout `/route-planning`: "Looking near {location}" in the left rail, "People, groups, and places near {location} can help make it real" in the route list intro, "Local openings that match this route, near {location}" in the opportunities section, and a "Near {location}" tag on every opportunity card.

## Local opportunity language

- Not an address — a free-text, editable city/state field
- No browser geolocation is used
- Default: Austin, TX

## Route mapping logic

Question 4 ("What would make this feel more reachable?") maps to a suggested route:

| Answer | Route |
|---|---|
| Someone who has done it before | People Route |
| A class, event, or opening | Real Openings Route |
| A group or community | Community Route |
| A breakdown of requirements | Requirements Route |
| A small low-pressure trial | Try It Route |
| I'm not sure — suggest one | Real Openings Route |

Any other (custom, freeform) answer also falls back to Real Openings Route, so the default demo and any custom text still produce a sensible starting point. See `mapReachableToRouteId` and `getRouteSuggestionReason` in [`lib/direction.ts`](../lib/direction.ts).

## Mock opportunity examples

One opportunity per route, in [`lib/opportunities.ts`](../lib/opportunities.ts):

| Route | Opportunity | Type | Purpose |
|---|---|---|---|
| People Route | Talk with Someone Who Made the Shift | Person / Conversation | Learn what actually helped |
| Requirements Route | Beginner Setup Checklist | Planning / Requirements | Lower friction and know what you need |
| Real Openings Route | Plant-Based Cooking Class | Class / Event | Try the change in a real setting |
| Community Route | Beginner Vegetarian Meetup | Group / Community | Be around people already living this |
| Try It Route | 7-Day Plant-Forward Trial | Small Trial | Test the path before committing |

Only Plant-Based Cooking Class is clickable, linking to the existing `/opportunity/plant-based-cooking-class` page. The other four are marked "Preview" and are not yet clickable — they are mock content only, not real opportunity pages.

## Posting preview principle

Posting an opportunity is not live. The preview modal makes that explicit: *"Opportunity posting is coming soon. Pathoro will help people, groups, mentors, teachers, businesses, and communities place opportunities where they become meaningful and reachable."* No submissions are collected — the preview fields are disabled placeholders, and the modal only offers a "Got it" close action.

## What is still mocked

- All opportunity, route, and profile content lives in static files in `lib/*.ts`
- Location is a free-text field with no geocoding, validation, or real distance calculation
- Only one opportunity (Plant-Based Cooking Class) is a real, clickable detail page — the rest are preview-only mock cards
- "Post an opportunity" is a preview only — no submissions are collected, no backend exists
- Answers and location persist only in the current browser's `localStorage`
- No authentication, accounts, real AI, or recommendation logic
