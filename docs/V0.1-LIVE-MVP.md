# Pathoro v0.1 — Live MVP

**Live URL:** https://pathoro-mvp.vercel.app

## Production status

- V0.1 is live on Vercel
- Three-screen flow is working
- Production CTA navigation has been verified

## Current screens

1. Onboarding / Landscape — welcome, orientation, and "You Are Here" opportunity map
2. Route Planning — the five approved route families, with route detail and next steps
3. Opportunity Detail — a single example opportunity (Plant-Based Cooking Class) reached via the Real Openings Route

## Current routes

| Route | Screen |
|---|---|
| `/` | Onboarding / Landscape |
| `/route-planning` | Route Planning |
| `/opportunity/plant-based-cooking-class` | Opportunity Detail |

## What works

- Full click-through flow: `/` → `/route-planning` → `/opportunity/plant-based-cooking-class` → back to `/route-planning`
- Five route families displayed and selectable on Route Planning, with Real Openings Route as the default
- "Take this step" leads to a real opportunity detail page
- Buttons like "Take this opportunity," "Save for later," "View class options," "Join waitlist instead," and "Reflect after" show simple mock clicked states
- Production build confirmed clean: no 404s, no console errors, visuals verified against approved design

## What is mocked

- All opportunity, route, and profile content lives in static files in `lib/*.ts`
- There is no real user, no saved state between sessions, and no dynamic data source

## What is intentionally not built yet

- Authentication / accounts
- A real database or backend
- Real AI / recommendation logic
- Opportunity posting tools (for people, groups, businesses, mentors, organizers)
- Payments
- Mobile-specific layout polish

See [`MVP-LOCKED-PRINCIPLES.md`](MVP-LOCKED-PRINCIPLES.md) for the product principles this MVP is designed around but does not yet implement.
